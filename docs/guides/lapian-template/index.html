拉片模板：短剧和长视频拆解表怎么做

太长不看

一份实用拉片模板至少要记录三层信息：镜头事实（时间码、景别、运镜、画面）、声音事实（角色、台词、OS/VO、音效）、叙事判断（情绪、冲突、钩子、叙事功能）。事实字段越客观，后续分析越可靠。

拉片表的最小字段

字段 怎么填 为什么重要
镜头号 从 1 开始递增，或沿用自动切镜 ID 让每条分析能回到原始镜头
起止时间 00:01:23 → 00:01:27 计算节奏、场均时长和钩子位置
景别 / 运镜 近景、特写、推、拉、摇、跟等 观察情绪强调和信息释放方式
角色 本镜头出现或说话的人物 分析人物关系和出场密度
台词 原声逐句记录，不润色、不改写 投稿、复盘和二次创作都依赖原始台词
画面描述 客观写发生了什么，不先写观点 把视觉信息从“感觉”变成可比较数据
情绪 紧张、压抑、反转、爽点、失落等 形成情绪曲线
叙事功能 开场钩子、信息补充、冲突升级、转折、收束等 判断每个镜头为什么存在

推荐 CSV 表头

如果你手动做拉片，可以直接从这组表头开始：

镜头号,起始时间,结束时间,时长(s),景别,运镜,场景ID,场景地点,角色,台词,画面描述,情绪,宏情绪,叙事功能

积米律动导出的 breakdown.csv 也使用类似结构。区别是它会从视频自动切镜、转写台词、标注情绪和叙事功能，再输出给你人工复核。

短剧拉片要额外盯什么

  • 首个钩子出现时间：短剧通常在前几十秒内必须给出足够强的信息差、冲突或爽点。
  • 每集结尾钩子：结尾是否留下足够明确的“下一集理由”。
  • 冲突升级节奏：冲突是持续升级，还是中途泄气。
  • 角色关系变化：身份揭示、误会、反转、联盟与背叛是否有清晰节点。
  • 情绪密度：爽、虐、悬疑、喜剧等情绪是否分布稳定。

不要把观点混进事实字段

好的拉片表要先客观，再判断。比如“女主转身离开，男主停在门口没有追”是画面事实；“男主在逃避责任”是叙事判断。两者都重要，但应该放在不同字段里。字段分清后，同一部剧才能被不同人复核，也才能被 AI 或数据库继续统计。

AI 拉片适合替代哪一段劳动

AI 拉片最适合替代“记录性劳动”：切镜、抄台词、整理镜头表、初步标注情绪和叙事功能。人的判断仍然放在最后：这个钩子为什么有效，这条情绪曲线为什么能留住观众，哪些结构值得学习，哪些只是题材红利。
